text

text-align
- justify
-- 마지막 라인을 제외하고 다른 라인들의 줄바꿈이 일어날 떄 공백을 조절하여 자연스럽게 줄바꿈이 일어나게 해줌
- 블록 레벨 요소 안에 있는 인라인 혹은 인라인 블록 요소에만 적용

마진은 요소의 넒이에 포함되지 않음
- 요소 사이의 간격

인라인 요소는 패딩, 좌우 마진만 적용됨
- 상하 마진, 높이, 넓이는 조절되지 않음

라인 하이트에서 폰트 사이즈를 제거하고 하프 리딩을 위아래에 적절히 부여함

컨텐츠의 크기에 따라 요소의 넓이가 조절될 수 있도록 자연스럽게 두는 편이 좋음
- 화면에 넓이에 따라 레이아웃이 유연하게 변하면서 깨지지 않게 만드는 것이 중요

세로 중앙 정렬 방법
- 라인 하이트를 부모의 하이트와 맞추기
-- 여러 줄 일 경우 적용되지 않음
- position relative에 top으로 비율 조정
- transtorm 사용
-- 아직 배우지 않음

vertical align
- 인라인 요소의 수직 정렬을 맞추기 위해 사용하는 속성
-- 블록 레벨 요소나 인라인 블록 요소에 적용되지 않음

라인 하이트 1
- 하프 리딩을 제거한다

부모의 높이는 자신의 컨텐츠의 높이에 종속됨
- 이미지를 적용했을 때 아래에 약간의 공간이 남음
-- 이미지의 display 속성이 인라인이기 때문에 발생
-- 이미지도 텍스트로 인식이 되고, baseline을 가지고 있는 것처럼 인식됨
-- vertical align top이나 bottom을 주면 베이스 라인이 제거됨

text indent
- cm, mm등의 단위도 지원

text-decoration
- 폰트 텍스트의 형태에 따라 잘리는 경우가 있기 때문에 스타일링의 용도로 잘 사용하지는 않음
- a 태그의 기본 스타일을 제거할 때 많이 사용

text overflow
- text-overflow 속성은 그 자체만으로 넘친 컨텐츠를 만들어서 처리하지 않습니다.
- 컨테이너에 overflow:hidden, white-space:nowrap 속성이 같이 사용되어야 합니다.
- white-space:nowrap
-- 공백을 감싸지 않겟다.

아직 여러 줄 말줄임 코드가 표준이 없음
- 말줄임 같은 것들을 융통성 있게 해야한다

position

포지션을 통해 페이지의 레이아웃을 결정할 수 있다

기본은 static
- html 파일에 쓴 태그 순대로 배치됨

relative
- 원래 자신이 있어야 하는 위치에서 상대적
- static 속성일 경우에는 top, left, right, bottom 같은 속성이 적용되지 않음
- 다른 컨텐츠들의 레이아웃에 영향을 미치지 않음

absolute
- absolute는 static을 제외한 position 속성값을 가진 가장 가까운 부모의 박스 내를 기준으로 위치
- relative, absolute, sticky
- 부모에 static이 있으면 계속 거슬러 올라감
- 다른 요소에서는 찾을 수 없음
-- 있는지 없는지 모름
- top, left 를 0하면 붙는 이유
-- 패딩

피그마는 폰트 정보가 중요
- 나머지는 적절하게

fixed
- 스크롤을 올리거나 내릴때 고정
- 화면(뷰포트)에 고정

sticky
- 조상에 스크롤이 있다면 가장 가까운 부모 요소의 컨텐츠에 들러붙음
- 스크롤 필수

z-index
- 요소들끼리 겹칠 때 어떤 요소가 앞에 나타날 것인지를 정함
- static인 경우 제외
- 마이너스 값도 줄 수 잇음
-- 1만 사용
- 부모가 자식 앞으로 나올 수는 없지만 자식이 부모 뒤로 갈 수는 잇음
-- 자식의 z인덱스는 부모의 z인덱스를 기준으로 결정됨??
- 값이 동일하면 만들어진 순서대로 겹쳐짐
- 부모가 다른 요소보다 값이 작을 경우 자식의 값이 아무리 크더라도 해당 요소를 이길 수 없음

float
- 띄우다
- 요소를 띄워서 좌 혹은 우로 정렬하는 속상
- 텍스트들은 floating된 요소들의 위치를 알아본다
- floating 되지 않은 다른 요소들은 알아보지 못함
- text-align의 경우 인라인이나 인라인 블록만 정렬가능
-- 블록 레벨 요소들 정렬에 사용
--- 가지고 있는 컨텐츠의 크기 만큼만 점유

- 인라인 요소에 float 적용하면 자동으로 block 레벨 요소로 바뀜

자식들이 모두 float 속성을 가질 때 컨테이너 요소가 자신의 높이에 자식 요소들의 높이를 반영하지 못한다
- 부모 요소에 overfolw visible을 제외한 속성 추가
- 부모 요소에 높이 지정
- clear 속성 사용
-- float를 해제하기 위해 사용
-- 다른 요소들이 float을 알아보게 만든다
-- 가상 요소를 이용하여 별도의 클래스를 생성하지 않아도 됨
- 부모에 display inline-block 적용


inline요소들은 float된 요소들을 알아본다
- 태그가 알아보는 것이 아님
