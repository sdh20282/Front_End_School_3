각자만의 페이스를 유지해야함

######################
flex

flex basis
- basis는 항상 넓이가 아니다
-- 주축에 따라 넓이가 되기도 하고 높이가 되기도 한다.
- width와 같이 있을 때 basis의 값이 우선됨
-- width는 넓이 고정, basis는 컨텐츠 양에 따라 늘어날 수도 잇음


flex grow
- 자라다
-- 0이 초기값
- 가용 가능한 넓이만큼 item의 넓이가 늘어남
-- 모두 다 grow를 주면 늘어날 수 있는 공간이 있다면 그 공간을 모두 동일하게 차지
-- 특정 아이템에만 grow를 주면 늘어날 수 있는 공간이 있다면 해당 아이템이 그 공간을 모두 차지
-- 값을 다르게 줄 경우 넓이가 아닌 컨텐츠 넓이를 뺀 여백의 넓이를 비율대로 나눠가짐

flex-shrink
- flex-grow에 반대되는 개념
- 컨테이너의 공간이 줄어들때 flex-basis의 값에서 더 줄어들어도 되는지 지정하는 값
- 크기를 고정하거나 줄이는 역할
- 0을 사용할 경우 컨테이너의 크기가 줄어도 값은 고정됨
-- 더 줄어들지 않게 하기 위해 사용

basis, grow, shrink 한꺼번에
- flex 속성
-- 축약 속성 사용을 권고

align-self
- 부모의 align-items 속성을 덮어 flex-item 에게 개별적인 align-items 속성을 부여
- 기본값은 align-items와 마찬가지로 stretch 입니다.

order
- flex-item 들의 순서를 결정
- 수의 의미가 아니라 수의 크기로 순위 결정
-- 수가 작을 수록 더 우선 순위 (음수 가능)
- 접근성 때문에 많이 사용
-- 스크린 리더는 순서대로 읽지만 보여지는 것은 앞에 있어야 함??

gap
- flex에서 아이템 사이 간격

#############################
grid
- 부모 컨테이너 요소안에서 내부 자식 요소들의 위치를 X축과 Y축 방향 모두를 이용해 배치하는 내부디스플레이타입

fr
- fraction (분수, 일부, 비율)
- grid 컨테이너안에서 사용되는 데이터 타입으로, 상황에 따른 유연한 길이를 의미

grid-template-columns
- 열방향 그리드 트랙의 사이즈를 설정

grid-template-rows
- 행방향 그리드 트랙의 사이즈를 설정

repeat() 함수
- row 혹은 column 방향으로 grid-track 의 사이즈를 좀 더 간단한 형태로 표현하도록 도와주는 CSS 함수
- 첫번째 인자는 반복 횟수, 두번쨰 인자는 반복할 값

minmax() 함수
- 최소와 최대 사이의 범위를 설정하는 함수
- 최소, 최대 넓이 지정할 때 사용

auto-fill & auto-fit
- repeat 함수를 사용할 때, 반복되는 카운트를 고정하지 않고 컨테이너의 넓이에 따라 가능한 많은 그리드 컬럼을 배치하고 싶다면 사용하는 키워드 값
- auto-fill의 경우 더 채울 수 있을 때 채우지 않음
- auto-fit의 경우 채울 수 있으면 채움
- fill은 min에, fit은 max에 기준..?

gap
- 셀과 셀 사이의 간격을 설정

그리드 아이템의 영역(grid area) 설정
- 행 방향 : 행 방향의 벽을 부숨
- 열 방향 : 열 방향의 벽을 부숨
- 칸을 나누고 병합하는 것이 아니라 grid-area 속성을 이용하여 영역의 이름을 지어주고, grid-template-areas 속성을 통해 해당 영역의 범위를 시각적으로 표현할 수 있음
-- none 대신 . 사용 (1칸밖에 안비워짐)

transition / transform

gpu등과 같은 것들을 사용할 때에는 명시해주어야 함

transition
- CSS 속성값이 변할 때, 값의 변화가 일정 시간에 걸쳐 일어나도록 하는 것

transition-delay
- transition 이 일어날 때 얼마만큼 기다렸다가 실행할지 결정해주는 속성

transition-duration
- transition이 일어나는 지속시간을 설정

transition-property
- 어떤 속성에  transition효과를 줄지 설정

transition-timing-function
- transition의 진행 속도를 설정

trainsition
- 위의 내용을 한번에 설정


transform
- 정말 많이 사용
- html 요소를 여러 형태로 변형할 때 사용하는 속성으로, transition과 함께 사용하면 엄청난 효과를 낼 수 있는 속성

scale
- 기준점은 center
-- 중심을 기준으로 커짐
-- 변경 가능
-- 다른 기준점은 왼쪽위

rotate
- deg, turn

translate
- 좌표 이동
- x, y 따로 지정 가능
- 이동했을 때 hover되는 기준은 원래 위치 기준

skew
- 기울이기

transform-origin
- 어떤 기준점을 잡게 할 것이냐
- 어디를 기준으로 움직이게 할 것이냐

translate vs position
- 단순 배치
-- position
- 애니메이션이나 혹은 동적으로 요소의 위치를 이동해야 하는 경우
-- transform
